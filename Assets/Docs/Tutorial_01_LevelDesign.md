# Tutorial 01 Level Design: 锈铁维修站

## Goal

让玩家在 5-8 分钟内完成第一张正式地图：小机器人从废弃维修站醒来，学习移动、奔跑、跳跃、攻击、交互、观察机关节奏，击败「维修站守卫者」后进入机械城入口。

## Beat Layout

1. 出生维修台：约 10 秒可跳过电影感叙事片头，先用居中琥珀单色工业启动终端逐行显示 A-07 故障日志，再慢速扫过废弃维修站，最后回到破旧维修床播放更完整的苏醒扫描光、短动画维修臂、断线和红色状态灯；随后教学 A / D、Shift、Space、E。
2. 移动教学区：低矮障碍、小台阶、右向灯条和可破坏箱；教学普通移动、奔跑、跳跃和 J 攻击。
3. 平台跳跃区：三段高低平台，中间废油/安全回传区；训练跳跃落点。
4. 第一个敌人区：低威胁巡逻维修机器人，足够空间给玩家练习攻击和躲避。
5. 陷阱机械厅：间歇蓝色电流地板与上下压缩机，红灯预警，训练观察节奏。
6. 充电存档点：临时充电桩，按 E 恢复生命并记录检查点。
7. Boss 维修大厅：玩家进厅后触发维修站守卫者遭遇，入口红灯锁闭、Boss 血条出现、镜头临时收紧；P1 读横扫/砸地，P2 半血召唤小维修机并加入冲击波、过载电弧、核心光束与顶部落雷，P3 低血核心暴走并加入终段连携。
8. 机械城入口门：Boss 死亡后 `Door_BossExit_MechCity` 自动开门，玩家进入下一张地图方向。

## Tuning Defaults

- 玩家行走速度：4.1
- 玩家奔跑速度：7.2，按住 Shift 启用
- 地面加速度：44
- 地面减速度：58
- 空中加速度：30
- 跳跃力度：13
- 重力：3.4
- 土狼时间：0.12 秒
- 跳跃缓冲：0.12 秒
- 普通攻击冷却：0.28 秒
- 普通攻击范围：1.1
- 可破坏铁箱生命：2，高度为原铁箱的 3 倍，要求玩家用攻击打碎后通过
- 第一个巡逻维修机器人生命：3
- Boss 生命：14
- Boss 伤害：每个 Boss 招式 1 点，玩家默认 3 耐久
- Boss 二阶段：7 HP 触发，只召唤一次 2 台 1 耐久小维修机器人，之后加入贴地冲击波、过载电弧、核心光束和顶部落雷
- Boss 三阶段：4 HP 触发核心暴走，V4 破损覆盖层显现，加入双向核心脉冲、横扫接冲击波、砸地接电弧连携
- Boss 收招窗口：横扫约 0.55 秒，砸地约 0.75 秒，冲击波约 0.85 秒，过载电弧约 0.78 秒，核心光束约 0.86 秒，顶部落雷约 0.88 秒，终段连携约 0.95 秒
- Boss 死亡演出：约 1.8 秒，核心爆闪、外甲裂纹、碎片飞散、烟雾火花喷出，最后本体淡出隐藏
- 复活电影感：约 1.25 秒，0.38 秒左右传送到当前检查点，期间锁定输入并恢复耐久。

## Art Panel Layout

`BackgroundPanorama_RustRepairStation_v15_*` 使用 5 张连续横版切片，统一透视、地平线、光源和色调。V15 背景降低噪声和对比，主角活动高度保持干净；切片交界处用暗柱、雾带和大结构遮挡，避免出现明显竖向分界线。V8/V10 仍负责可踩道路语言，V15/V14/V13 负责后景、中景和高空前景动态装饰，`BG_SystemPolish_Readability` 负责最终可读性通刷，`BG_DynamicPolish_V16` 负责清晰增强的区域动态呼吸。

1. x 约 0-18：出生维修台，暖橙维修灯、单一主吊灯、状态灯和低透明维修墙面。
2. x 约 18-40：移动教学，加宽平地、加高低障碍、可破坏箱、小台阶。
3. x 约 40-66：破损跳跃通道，三段清晰落点、废油回传、蒸汽和高空吊轨。
4. x 约 66-86：第一个敌人区，开阔战斗地面、警示灯、机器人残骸。
5. x 约 86-112：陷阱机械厅，缩短电流跨距、绝缘踏板、压缩机和蓝白电弧。
6. x 约 112-126：临时充电站，绿色能源灯和低噪声安全背景。
7. x 约 126-160：Boss 维修大厅，宽阔地面、Boss 门、粗管、蒸汽和慢转齿轮。
8. x 约 160-176：机械城入口门，出口大门和较安静的远景门框。

## Detail Pass Notes

- 所有区域细节组命名为 `Details_XX_*`，可在 Hierarchy 中独立折叠和替换。
- 主要可踩平台由隐藏碰撞体和 V8/V10 精细视觉层组成：暗钢板、琥珀上沿、前侧阴影、接缝、铆钉、划痕和破损角；当前精修只收细边线、压低噪声、增强侧面层次，不新增道路 PNG。
- 红色闪烁元素必须使用 V7 警示灯 sprite，不再允许无语义的小红点或小方块。
- V7 环境组件负责维修台、机械臂、管道组、铁链、齿轮墙、灯具、箱子、压缩机、充电桩、Boss 门、入口门、机器人残骸和平台组件。
- Effects/V2 负责蓝色电流地板、蒸汽、火花、灯光 halo、落灰雾层、扫描光和 Boss 砸击尘浪。
- Effects/V1 继续负责玩家攻击三段弧、命中火花、跑步尘土、起跳火花和落地尘雾。
- Enemies/V2 负责巡逻维修机器人、红眼、损坏残骸和维修站守卫者基础身体；Enemies/V3 只提供 Boss 精细外甲覆盖件。
- `SteamPuff2D` 用在破损管线和 Boss 厅蒸汽口；`SimpleRotator2D` 用在远景风扇；`LoopingBackgroundMotion2D` 用在火花与扫描光。
- `BG_DynamicDecor_V13` 补充更明显的三类动态：电气闪电、机械旋转/传送带光带、灰尘/蒸汽。所有 V13 装饰无碰撞，近景层不做视差移动。
- 路灯只保留 V11 道路层版本，V13 不再重复生成道路后方路灯。
- 开局只保留 `MidDecor_V11_Lamp_Awakening` 一盏主吊灯；V8、V9、V13 的开局重复灯源不再生成，避免开局出现两个灯并排或重叠。
- `BG_LargeDecor_V14` 使用用户提供的大图作为大尺度背景主题件：远景机械墙和桥景，中景风扇、齿轮、锅炉、机械框架，近景高空吊轨和吊钩。开局上缘增加零视差吊轨和吊钩，第一张大电缆桥图不再放入场景。V14 只放在道路之后或上方，动态包括吊钩摆动、风扇/齿轮慢转、电弧、蒸汽、闪烁灯和浮尘。
- `BG_MapReadableDecor_V15` 是当前可读性优先的大型动态层：远景放低透明机械墙/锅炉/传送带/齿轮墙，中景放风扇、电弧、蒸汽、灯光和扫描光，近景只放高空吊轨和吊钩，且近景层零视差。
- `BG_SystemPolish_Readability` 不新增碰撞，不使用道路琥珀可站立边线；它只用路线暗带、区域地标柔光、危险/交互色彩和少量高空氛围，把八个教学区域的读图语言统一起来。
- `BG_DynamicPolish_V16` 不新增碰撞、不新增 PNG，分为 Far、Mid、NearSafe、GameplayReadable 四层；远景用尘雾/传送带/慢转机械，中景用区域灯光、电弧、蒸汽和扫描光，近景只放高处链条/旗帜，低位可读层只放地面下方柔雾/柔光，不能压住玩家、平台边线、陷阱和互动物。
- `*_PolishRefined` 是当前组件精细化层，只挂在记录点、日志终端、电流地板、电闸、门、充电站和补给箱附近；V19 透明覆盖件提供终端屏幕、控制/门锁板和状态核心，现有 halo/scan/electric/steam 负责动态反馈。所有节点无碰撞，琥珀代表待机，青绿代表安全/补给，红灯代表危险/锁定，蓝白代表电流。
- `*_FXPolish` 是组件特效增强层，只挂在 Boss 入口锁、Boss 出口门、记录点、移动区箭头和出口箭头附近；它复用现有 halo、scan、spark、dust 素材强化状态反馈，全部无碰撞，不新增图片，不改变路线或触发器。
- `Environment/V20` 是 Boss 战门体精细覆盖件批次，只服务入口锁门和 Boss 出口门：入口用红色锁芯、液压夹具和电弧表达锁闭，出口用青绿门缝、双层门框和蒸汽表达通行，碰撞和开门逻辑仍由原隐藏逻辑层负责。
- `AwakeningBench_RefinedAssembly` 本体使用 `Environment/V12/spawn_repair_bed_refined.png`；`AwakeningBench_TablePolish` 只挂在出生维修台细节组下，台体磨损、铆钉、油污、底部暖光、线缆、扫描线和状态灯都服务于“维修台苏醒”主题；不再把这类美化扩散到主路平台。
- `SpawnIntro_AwakeningPolish` 只覆盖出生维修台开场，不新增碰撞；黑屏日志使用居中琥珀单色工业终端、清晰字号、逐行显示、低噪声扫描线和进度条交代 A-07 失忆苏醒，环境扫镜从废弃维修站暗处回到维修台，暖 halo、低雾、顶部细尘、服务面板微光、扫描光、蒸汽、火花、少量蓝白电弧、台体微震和短动画维修臂都在玩家轮廓后方或边缘，保证向右路线清楚。
- `JumpRoute_BackgroundPolish_BrokenPlatforms` 专门覆盖跳跃教学路段的后景：下层平台后方用暖 halo，右侧平台用青色柔光提示路线，下方油雾和蒸汽只在平台下方活动，高处链条轻摆，背景机械墙和破窗压低透明度，保证平台、箱子、玩家和右侧落点清楚。
- `RespawnPolish_Runtime` 负责所有失败回传的系统重组动画，`RespawnPoint_Polish_AwakeningBench`、`RespawnPoint_Polish_BeforeJump`、`RespawnPoint_Polish_ChargingStation` 只做检查点柔光和激活脉冲，不新增碰撞、不改变检查点位置。
- Boss 战专用逻辑集中在 `BossEncounterController2D` 和 `RepairStationBoss2D`：前者只管触发、入口锁、Boss 血条和临时镜头边界，后者只管 dormant/intro、读招方向、三阶段、召唤、冲击波、过载电弧、核心光束、顶部落雷、终段核心脉冲、连携和破碎消失。
- Boss 横扫、砸地、冲击波在 windup 时锁定玩家所在侧，移动 hitbox 和尘浪/冲击波视觉；过载电弧锁定玩家脚下局部位置，核心光束先给低位扫描线，顶部落雷先给三段竖向警示柱。P3 终段核心脉冲先给红橙低位预警，随后蓝白电流向两侧爆发；横扫接冲击波和砸地接电弧都保留约 0.95 秒反击窗口。
- HUD 使用废土工业终端 IMGUI 游戏界面，目标固定在左上安全区，耐久、Boss 血条、提示和交互文本分层显示，以小字号、短文案、暗钢锈铁面板、琥珀/青绿/红蓝状态灯、机械分段条、轻量扫描线和受击闪烁为主，避免 1280x720 下越界或遮挡动作；镜头保留纵向稳定，并增加轻微横向预判以减少奔跑时的压迫感，Boss 战时通过临时边界框住维修大厅。
- 主角通过 `Boxbot_Rim_*`、`Boxbot_GroundShadow`、眼灯和天线灯从棕黑背景中分离。
- 背景装饰层保持有序：顶部主管线、中层设备、底部箱体三条轨道；每屏 3-5 个主要视觉物件，不把细节放到相机实际经过范围之外。
- 新验证器会禁止当前场景出现可见 `white_pixel` SpriteRenderer；纯色块只能作为隐藏碰撞或内部备用资产存在。
